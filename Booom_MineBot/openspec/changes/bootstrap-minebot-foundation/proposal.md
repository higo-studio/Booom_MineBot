## 背景

当前仓库只是一个完成基础包配置的 Unity 6 空工程，还没有玩法代码、项目结构、实现顺序或 Agent 工作约定。策划案已经明确了 GameJam 版本的核心循环是“方格挖掘 + 扫雷式风险判断 + 从属机器人自动化 + 地震波生存压力”，因此必须先把技术基础框架和开发顺序定下来，避免后续实现各自为战。

## 变更内容

- 为 BOOOM Minebot 定义 Unity MVP 的工程基础框架，包括目录布局、程序集边界、启动场景流程、测试脚手架和配置资产归属。
- 明确第一版运行时能力切分，让核心循环按系统拆分实现，而不是堆进单一场景逻辑。
- 确认 MVP 关键技术栈：Unity 6、C#、URP、Input System、UGUI、ScriptableObject 配置资产、Unity Test Framework。
- 显式延后非 MVP 技术方向，例如 DOTS/ECS、多人联机、复杂指令型机器人模式和重型内容流式加载方案。
- 初始化仓库内的 Agent 项目 skill，使后续任务统一沿用飞书策划源、OpenSpec 流程和项目级 Unity 约束。
- 产出按依赖排序的执行任务清单，使开发流程从项目基础逐步收敛到可玩的多波次循环。

## 能力范围

### 新增能力
- `project-foundation`：建立项目结构、程序集布局、启动场景、测试脚手架，以及仓库内的 Agent 技能约束。
- `grid-mining-loop`：支持方格地图、玩家移动、岩壁交互、硬度门槛、挖掘和资源/经验掉落。
- `hazard-inference`：支持隐藏炸药分布、探测数字反馈、标记状态以及爆炸对地形的结算。
- `progression-and-base-ops`：支持经验成长、即时升级 UI、生命/维修流程和 MVP 资源/建造经济。
- `automation-and-wave-survival`：支持从属机器人自动挖掘、地震波危险区、波次计分和失败条件。

### 修改能力
- 无。

## 影响范围

- 在 `Assets/Scripts` 下新增运行时模块、启动逻辑、UI 表现层和测试代码。
- 新增或调整 Unity 资产，包括启动场景、ScriptableObject 配置、与挖掘/探测/维修/机器人/波次相关的预制体。
- 更新 `openspec/` 和 `.codex/skills/`，使规划、实现和 Agent 工作流共用同一套项目约束。
- 本次变更不涉及外部服务或 API，只涉及项目内部基础框架和玩法脚手架。
