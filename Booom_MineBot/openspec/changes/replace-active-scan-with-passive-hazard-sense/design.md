## Context

当前实现里，探测由以下旧链路驱动：

```text
Q / HUD 按钮
-> GameplayInputController.ScanCurrentCell()
-> GameSessionService.Scan(origin)
-> HazardService.ScanFrontierWalls(...)
-> MinebotGameplayPresentation / ScanIndicatorPresenter
```

这条路径有两个问题：

- 风险信息必须靠额外按键触发，和当前“移动即挖掘”的节奏不一致。
- 探测消耗能量，导致“周边危险感知”既是输入操作又是经济消耗点，不适合作为持续存在的被动能力。

与此同时，当前挖掘规则还是：

- 安全墙只会被单格挖开
- 炸弹墙会触发 `ResolveExplosion`，把半径范围内的一片墙直接清空

这让“扫描数字”并没有真正成为开格规则的一部分，也使炸弹的惩罚更像地形炸穿而不是单次踩雷。

这次变更要同时完成两件事：

- 把探测改成角色固有的周期性被动感知
- 把挖掘改成按扫描规则驱动的扫雷式连锁开格

## Goals / Non-Goals

**Goals**

- 把主动 `Q` 探测改为固定时间间隔的自动危险感知。
- 保留“只显示玩家附近、且贴着空地的前沿岩壁”的候选筛选规则。
- 保留每块候选岩壁自身 `3x3` 范围炸药计数与数字浮标表现。
- 让安全墙的开启规则与 `3x3` 炸药计数含义一致，形成扫雷式连锁破碎。
- 把炸弹命中改成“只炸开当前格并扣血”的单次惩罚。
- 删除主动探测输入、提示文案和按钮语义。
- 让被动感知不再消耗能量，并补齐对应回归测试。

**Non-Goals**

- 不重做炸药生成规则、标记规则、危险区规则或机器人自动化边界。
- 不改扫描数字的视觉风格、对象池或世界坐标挂点。
- 不在本轮移除能量资源本身，只移除它与探测之间的消耗关系。
- 不把被动感知扩展成全地图持续透视；它仍只覆盖玩家附近的前沿岩壁。
- 不把安全连锁开格扩展成“整片非炸弹墙一次清空”；扩散仍受 `3x3` 风险值约束。

## Decisions

### 1. 用周期性被动刷新替代主动 `Scan()` 调用

`GameSessionService` 不再暴露“主动执行一次探测并返回成功/失败”的主路径，改为提供：

- 立即刷新一次当前玩家周边危险感知
- 按配置时间间隔自动刷新一次当前玩家周边危险感知

刷新输入源固定为 `player.Position`，不再接受 UI 或输入层传入的任意 origin。

### 2. 候选岩壁筛选与计数规则保持不变

为了把玩法变化控制在“触发方式”而不是“危险判断公式”上，本轮继续复用：

- `HazardService.ScanFrontierWalls(playerPosition, frontierRange)`
- 前沿岩壁判定：可挖岩壁、至少相邻一个空地、离玩家处于邻近范围内
- 计数口径：以目标岩壁为中心的 `3x3` 炸药数

这样可以保证玩家看到的危险值含义不变，只是刷新方式从“按键触发”变成“自动刷新”。

### 3. 被动感知不再消耗能量

被动感知是角色固有能力，而不是一次性主动技能，因此本轮明确取消探测能量消耗。

实现上：

- `HazardRules.ScanEnergyCost` 暂时保留为兼容字段，但不再被运行时主路径消费
- `PlayerEconomy` 中能量资源继续存在
- HUD 仍可显示当前能量，但不会因危险感知自动下降

### 4. 保留扫描数字图层，但删除主动交互语义

`ScanIndicatorPresenter` 与现有数字表现继续保留，因为“周边危险值”仍然需要显示在目标岩壁上方。

但以下主动交互语义必须删除：

- `Q` 触发探测
- “按 Q 探测前沿岩壁数字”文案
- HUD 中把第 4 个主操作按钮当作主动探测按钮的行为

替代方式：

- HUD 文案改为“被动探测/周边感知”
- 如果保留第 4 个按钮槽位，则只作为状态标签，不再执行输入命令

### 5. 被动刷新事件不应污染普通反馈消息

主动探测时代，完成一次探测会刷新 `feedbackMessage` 为“探测完成：...”。  
改成周期性自动刷新后，如果继续沿用这条反馈路径，UI 会被自动提示刷屏。

因此本轮要求：

- 被动感知刷新只更新扫描数字和摘要状态
- 不再把每次自动刷新写进普通 `feedbackMessage`
- 普通反馈消息继续留给挖掘、拾取、建筑、机器人等玩家主动操作

### 6. 安全墙的连锁破碎应按 `3x3` 炸药计数驱动

为了让“扫描数字”直接对应开格逻辑，本轮将安全墙开启规则改成：

- 玩家或机器人命中一块非炸弹岩壁时，先按该岩壁为中心统计 `3x3` 炸药数
- 该数字大于 `0` 时，只破碎当前岩壁，不再继续扩散
- 该数字等于 `0` 时，继续向相邻安全岩壁扩散
- 扩散过程复用同一套 `3x3` 炸药计数规则，并在遇到“有数字的安全墙”时停止向外延伸

第一版扩散使用 `8` 邻接安全墙，这样和 `3x3` 风险判定是一致的，玩家看到的“零风险区域”也更接近传统扫雷的连通展开。

### 7. 炸弹命中改成单格惩罚，不再半径清图

当前 `ResolveExplosion` 会把命中炸弹周围一片区域一起清空，这和新的扫雷式开格节奏不匹配。  
本轮改为：

- 命中炸弹时，只把当前炸弹格炸开
- 玩家继续承受一次直接伤害
- 不再因为单次踩雷而额外清掉周围安全墙或触发连锁地形清图

这样炸弹的意义会更接近“踩错即受伤”，而不是“顺手帮玩家炸穿附近墙体”。

## Implementation Notes

推荐改动顺序：

1. 在规则层新增被动感知刷新接口与时间间隔配置。
2. 在规则层把单格挖掘改成“安全墙按数字连锁破碎 / 炸弹单格惩罚”。
3. 移除输入层对主动探测的订阅与调用。
4. 把表现层从“ScanCompleted + feedbackMessage”改成“被动感知更新 + 摘要刷新”。
5. 更新 HUD/按钮/提示文案。
6. 重写 EditMode / PlayMode 测试。

## Risks / Trade-offs

- [自动刷新后数字更新不再由玩家控制] -> 用固定时间间隔约束刷新频率，避免每帧跳动。
- [取消能量消耗会弱化能量资源用途] -> 本轮先接受能量用途收缩，后续再决定是否把能量转投给别的主动能力。
- [删掉主动探测后旧输入资产仍保留 `Scan` action] -> 允许输入资产先兼容保留，但运行时不得再订阅或响应该 action。
- [安全连锁开格可能一次性清掉过多墙体] -> 第一版只让 `0` 风险墙继续扩散，遇到有数字的安全墙立刻停下。
- [单格炸弹惩罚会削弱“爆炸改地形”的戏剧性] -> 本轮优先让炸弹与扫雷判断建立稳定关系，后续若需要范围破坏，可再单独定义特殊炸药或地震效果。
