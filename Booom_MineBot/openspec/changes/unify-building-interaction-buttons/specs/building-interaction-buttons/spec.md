## ADDED Requirements

### Requirement: 靠近可交互建筑必须显示建筑交互按钮
系统 SHALL 在玩家靠近可交互建筑时显示建筑交互按钮，并在玩家离开交互范围后隐藏对应按钮。第一版至少覆盖默认维修站和默认机器人工厂。

#### Scenario: 靠近维修站
- **WHEN** 玩家移动到维修站交互范围内
- **THEN** HUD 会显示维修站交互按钮

#### Scenario: 靠近机器人工厂
- **WHEN** 玩家移动到机器人工厂交互范围内
- **THEN** HUD 会显示生产从属机器人的交互按钮

#### Scenario: 离开可交互建筑
- **WHEN** 玩家不在任何可交互建筑的交互范围内
- **THEN** HUD 不会显示维修或生产从属机器人的建筑交互按钮

### Requirement: 维修站按钮必须执行维修流程
维修站交互按钮 SHALL 调用现有维修规则入口，按配置或现有参数消耗金属并恢复玩家生命值。维修结果 MUST 通过 HUD 反馈给玩家。

#### Scenario: 点击维修按钮且金属充足
- **WHEN** 受伤玩家靠近维修站并点击维修按钮，且金属资源足够
- **THEN** 系统会扣除维修成本、恢复玩家生命值，并显示维修完成反馈

#### Scenario: 点击维修按钮但金属不足
- **WHEN** 受伤玩家靠近维修站并点击维修按钮，但金属资源不足
- **THEN** 系统不会恢复生命值，并显示金属不足反馈

### Requirement: 机器人工厂按钮必须生产从属机器人
机器人工厂交互按钮 SHALL 调用现有机器人生产规则入口，按配置消耗金属并生成当前局内的从属机器人。生产结果 MUST 通过 HUD 和场景表现反馈给玩家。

#### Scenario: 点击生产机器人按钮且金属充足
- **WHEN** 玩家靠近机器人工厂并点击生产从属机器人按钮，且金属资源足够
- **THEN** 系统会扣除机器人成本、生成一个从属机器人实例，并在场景中显示机器人表现

#### Scenario: 点击生产机器人按钮但金属不足
- **WHEN** 玩家靠近机器人工厂并点击生产从属机器人按钮，但金属资源不足
- **THEN** 系统不会生成从属机器人，并显示金属不足反馈

### Requirement: 建筑交互按钮必须遵守高优先级输入锁定
建筑交互按钮 SHALL 在升级选择、GameOver、标记模式和建筑模式期间禁用或隐藏，避免建筑动作与其它 UI/地图模式冲突。

#### Scenario: 升级面板打开时靠近建筑
- **WHEN** 升级选择 UI 打开且玩家位于维修站或机器人工厂交互范围内
- **THEN** 建筑交互按钮不会执行维修或生产机器人动作

#### Scenario: 建筑模式打开时靠近机器人工厂
- **WHEN** 玩家处于 Build 模式且位于机器人工厂交互范围内
- **THEN** 生产从属机器人按钮不会执行生产动作

#### Scenario: 游戏失败后点击建筑交互按钮
- **WHEN** 游戏处于 GameOver 状态且玩家点击建筑交互按钮
- **THEN** 系统不会提交维修或生产机器人命令

### Requirement: 建筑交互按钮必须复用规则服务而不是复制结算
建筑交互按钮 SHALL 只转发交互意图到现有规则入口，不得在 UI 层复制资源扣除、生命恢复或机器人生成逻辑。

#### Scenario: UI 触发维修
- **WHEN** 玩家点击维修站按钮
- **THEN** 维修结算会通过现有维修规则入口完成

#### Scenario: UI 触发机器人生产
- **WHEN** 玩家点击机器人工厂按钮
- **THEN** 机器人生产会通过现有机器人工厂规则入口完成
